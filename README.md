# Bonheur Eternal

A single-page branding & lead-generation website for the luxury fragrance house **Bonheur Eternal**. Plain HTML, modular CSS, ES module JS. No frameworks. A tiny Python build step glues HTML partials together.

---

## Why plain HTML + a 70-line build script (not React/Vite)

This is a static, single-page brand site with:

- No dynamic data, no auth, no routing
- No server-side state
- A form that is wired to `mailto:` today and swappable to any endpoint later

React/Vite would add a build pipeline, a runtime, and a deploy target for no real benefit. Plain HTML ships instantly, loads faster, and deploys to any static host (Netlify, Vercel, Cloudflare Pages, GitHub Pages, S3, nginx) by uploading the folder. The only build step is a tiny Python script that expands `<!-- @include -->` directives so the source stays organized as partials.

## Running locally

Requires only **Python 3** (preinstalled on macOS).

```bash
make          # show available commands
make build    # regenerate index.html from src/
make serve    # build once and start http://127.0.0.1:5173
make dev      # watch src/ and rebuild on change + serve
make watch    # rebuild on change (no server)
make clean    # remove generated index.html
```

Change the port:

```bash
PORT=8080 make serve
```

If you prefer not to use `make`:

```bash
python3 build.py                          # build
python3 -m http.server 5173               # serve
python3 build.py --watch                  # watch
```

## Animations

### 1. Hero entrance (page load)

When the DOM is ready, `scripts/main.js` adds `.is-loaded` to `<body>`, triggering a cinematic reveal sequence driven entirely by CSS transitions in `styles/components/hero.css` and `styles/animations.css`:

1. The sand backdrop fades in while scaling down from `1.04 в†’ 1` (1.4s)
2. The bottle silhouette rises and fades in (starts at +0.25s)
3. A soft amber glow blooms behind the bottle (starts at +0.6s)
4. The two headline words stagger upward (0.45s, 0.60s)
5. The subtitle and CTA slide up (0.85s, 1.05s)
6. After all elements have settled, the bottle enters a gentle 7s infinite float

### 2. Scroll reveal (collection, about, contact)

Every element with the `.reveal` class starts slightly offset and transparent. `scripts/animations.js` uses a single `IntersectionObserver` to add `.is-visible` as each element enters the viewport. Stagger is controlled per-element via the CSS custom property `--reveal-delay`:

```html
<article class="card reveal" style="--reveal-delay: 200ms">вЂ¦</article>
```

The four collection cards cascade in at 0 / 100 / 200 / 300 ms, and every other section block uses the same pattern.

### 3. Reduced motion

If the visitor has `prefers-reduced-motion: reduce` set, all entrance animations and scroll reveals are short-circuited вЂ” content appears immediately. No JS runs the observer, no CSS transitions fire.

## Configuration points

### WhatsApp number

All "Order via WhatsApp" CTAs use the placeholder number `+1 (555) 123-4567`. Find-and-replace `15551234567` in `src/partials/` to go live, then `make build`.

### Contact form

The form uses a `mailto:` handoff вЂ” on submit, it opens the visitor's email client with the inquiry pre-filled (no backend required).

To upgrade to a real endpoint, edit `scripts/form.js` and replace the `window.location.href = mailto` block with a `fetch()` call to your service:

- **Formspree** вЂ” `fetch("https://formspree.io/f/YOUR_ID", { method: "POST", body: data })`
- **Resend / SendGrid** вЂ” call your own serverless function
- **Netlify Forms** вЂ” just add `data-netlify="true"` to the `<form>` in `src/partials/contact.html`

## Swapping in real images later

Everything is authored so real images drop in without layout shift.

### Hero background

In `src/partials/hero.html`, replace:

```html
<div class="hero__placeholder hero__placeholder--sand">
  <div class="hero__bottle" aria-hidden="true"></div>
  <div class="hero__glow" aria-hidden="true"></div>
</div>
```

with an `<img>` of the same visual role. The hero reserves the correct height (`min-height: 92vh`) either way.

### Product cards

Each card's `.card__media` maintains `aspect-ratio: 4/3`. Replace the inner `.card__placeholder` div with `<img src="assets/velvet-mist.jpg" style="width:100%;height:100%;object-fit:cover" />`.

### About

`.about__media` reserves `aspect-ratio: 4/5`. Replace `.about__placeholder` similarly.

## Design system

All tokens live at the top of `styles/tokens.css` in `:root`:

- **Colors** вЂ” warm sand, deep espresso, amber accents
- **Typography** вЂ” Cormorant Garamond (display) + Inter (body)
- **Spacing** вЂ” consistent scale `--s-1` through `--s-12`
- **Motion** вЂ” named easing curves and durations
- **Radii, shadows** вЂ” named tokens for reuse

To rebrand the entire site, edit only `:root`.

## Accessibility

- Semantic landmarks (`<header>`, `<main>`, `<section>`, `<footer>`)
- Visible focus rings on interactive elements
- `aria-label`, `aria-expanded`, `aria-live` on dynamic regions
- Respects `prefers-reduced-motion`

## Browser support

Modern evergreen browsers (Chrome, Firefox, Safari, Edge). Uses `clamp()`, `aspect-ratio`, CSS variables, `backdrop-filter`, ES modules, and `IntersectionObserver` вЂ” all widely supported.
